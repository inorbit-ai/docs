---
title: "Configuration Kinds"
description: "Reference documentation for all configuration object types in InOrbit"
---

InOrbit uses a declarative configuration model where you define **what** your robots should do, not how. Each configuration kind represents a specific aspect of your fleet setup—from data ingestion to incident alerts to custom dashboards.

Configuration objects follow a consistent structure:

```yaml
apiVersion: v0.1
kind: <KindName>
metadata:
  scope: account/<accountId>  # or robot/<accountId>/<robotId>
  id: my-config-id
spec:
  # Kind-specific fields
```

Use the [InOrbit CLI](/ground-control/configuration/cli) to apply, list, and manage configurations.
