InOrbit uses role-based access control to manage what users can see and do within the platform. Each user is assigned a role that determines their permissions.

## Default Roles

The following roles are pre-configured for your account:

| Role | Description | Typical Use |
|------|-------------|-------------|
| **Owner** | Full access to all features and settings | Account administrators |
| **Administrator** | Can manage users, dashboards, and configurations | IT/Operations managers |
| **Manager** | Can view all data and perform management actions | Team leads |
| **Engineer** | Technical access for troubleshooting and configuration | Robot engineers |
| **Operator** | Day-to-day robot operation and monitoring | Floor operators |
| **Viewer** | Read-only access to dashboards | Stakeholders, observers |

<Note>
  Administrator, Engineer, Operator, and Viewer roles are available in **Standard** and **Enterprise** editions.
</Note>

Different roles see different dashboards by default:

| Dashboard | Owner | Admin | Manager | Engineer | Operator | Viewer |
|-----------|:-----:|:-----:|:-------:|:--------:|:--------:|:------:|
| Locations | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| Fleet | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| Robot | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| Navigation | ✅ | ✅ | ✅ | ✅ | ✅ | ⚪ |
| Settings | ✅ | ✅ | ⚪ | ⚪ | ⚪ | ⚪ |
| Admin | ✅ | ✅ | ⚪ | ⚪ | ⚪ | ⚪ |

<Tip>
  Administrators can customize dashboard access per role based on your organization's needs.
</Tip>

## Managing Users

As an Owner or Administrator:

1. Go to **Settings** (click the three vertical dots menu in the upper right)
2. Select the **Admin** tab
3. Click **Users** in the left menu
4. Add, edit, or remove users and their roles
